# Contributing to YouTube Transcript Bot

Спасибо за интерес к проекту! Мы приветствуем вклад от сообщества.

## Как внести вклад

### Reporting Bugs

Если вы нашли баг, пожалуйста:

1. Проверьте, что баг еще не был зарепорчен в Issues
2. Создайте новый Issue с детальным описанием:
   - Шаги для воспроизведения
   - Ожидаемое поведение
   - Фактическое поведение
   - Версия Python, OS
   - Логи (если применимо)

### Предложение новых функций

1. Откройте Issue с описанием функционала
2. Объясните, почему это будет полезно
3. Дождитесь обсуждения перед началом работы

### Pull Requests

1. Fork проекта
2. Создайте feature branch (`git checkout -b feature/amazing-feature`)
3. Следуйте стилю кода проекта
4. Добавьте тесты для новой функциональности
5. Убедитесь, что все тесты проходят (`make test`)
6. Commit изменения (`git commit -m 'Add amazing feature'`)
7. Push в branch (`git push origin feature/amazing-feature`)
8. Откройте Pull Request

## Стандарты кода

### Python Style Guide

- Следуйте PEP 8
- Используйте type hints где возможно
- Максимальная длина строки: 100 символов
- Используйте docstrings для функций и классов

### Тестирование

- Покрытие тестами должно быть > 80%
- Каждая новая функция должна иметь тесты
- Используйте pytest для тестов
- Моки для внешних API

### Размер файлов

- Файлы не должны превышать 600 строк
- Разбивайте большие модули на несколько файлов
- Каждый модуль должен иметь одну четкую ответственность

### Commit Messages

Используйте понятные commit messages:

```
Add: новая функциональность
Fix: исправление бага
Update: обновление существующего функционала
Refactor: рефакторинг без изменения функциональности
Docs: обновление документации
Test: добавление или изменение тестов
```

## Локальная разработка

### Setup

```bash
# Клонирование
git clone <your-fork>
cd yt-transcript

# Виртуальное окружение
python -m venv venv
source venv/bin/activate

# Установка зависимостей
make install

# Копирование .env
cp .env.example .env
# Заполните .env своими ключами
```

### Запуск тестов

```bash
# Все тесты
make test

# С покрытием
make coverage

# Конкретный файл
pytest tests/test_youtube.py -v
```

### Запуск бота

```bash
make run
```

## Code Review Process

1. Автоматические проверки должны пройти
2. Минимум один одобряющий review
3. Код должен соответствовать стандартам
4. Все комментарии должны быть разрешены

## Вопросы?

Если у вас есть вопросы, создайте Issue или напишите в Discussions.

Спасибо за ваш вклад!
